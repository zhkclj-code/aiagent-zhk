### python基础
1. 可以搞一个虚拟环境给当前项目，也就是.venv，包含python环境、所需的lib包等
2. 对象.方法名(不带括号)，比如student.introduce，这是代表方法引用，而不是调用方法，这么写也不会报错
3. 所有实例方法，第一个参数都需要self，不然传参会有问题，调用时默认都会给当前调用对象传作第一个参数（隐式）
4. __new__和__init__，有些只有__new__有些只有__init__？一个是创建对象分配内存，一个是初始化实例，给属性赋值，只有__new__的一般不可变对象居多，比如str、tuple，一旦创建对象了，就不许修改属性，如果看似可变，实则内部是重新创建了新对象
5. 为什么容器类对象有时是通过自身.方法()调用，而有些则是通过统一的方法调用，比如len(x)，有时不知道哪些通过方法直接调用，哪些可以通过对象点出来？这是因为有些是该对象自身特有的方法，比如字典dict的items()，只有他才有，而像len()这是所有容器共有的特性，所以提出来单独作为一个内置方法，不需要每个容器类型都实现一遍，python有64个内置函数
   1. 一般内置函数返回新对象，而实例方法则在原对象修改
   2. 内置函数通用，实例方法特有
6. 在参数列表中/和*的作用是限位符，不接收参数，/保证前面都是位置传参不能用关键字传参，*之后只能用关键字传参，两个中间随便传
7. 字典和集合推导式都是使用{}，只是有没有:的区别，{}创建默认是字典，空集合只能set()；
8. 属性装饰器@property和@属性名.setter的方法名都得是属性名，属性需要避免同名冲突
   ```python
    @property
    def price(self) -> float:
        return self.price  # ❌ 调用自己 → 无限递归(会触发property拦截)
   ```
9.  如果setter中有值校验逻辑，在init中可以通过托管属性触发setter验证，比如self.price = price触发，真实的值存在self._price中
