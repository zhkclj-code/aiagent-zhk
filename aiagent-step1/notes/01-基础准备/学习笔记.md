## Day 1-2：Python 现代特性

### match-case 语句

**概念理解**：
- Python 3.10+ 新增的结构化模式匹配
- 类似 Java 的 switch-case，但功能更强大

**代码示例**：
```python
def handle_response(status: int) -> str:
    match status:
        case 200:
            return "成功"
        case 404:
            return "未找到"
        case 500:
            return "服务器错误"
        case _:
            return "未知状态"
```

**与 Java 对比**：
- Java switch-case：只能匹配常量
- Python match-case：支持模式匹配、类型匹配、解构

---

### 类型联合运算符

**旧写法**：
```python
from typing import Union

def process(data: Union[str, int]) -> str:
    return str(data)
```

**新写法**：
```python
def process(data: str | int) -> str:
    return str(data)
```

**优势**：
- 语法更简洁
- 可读性更好

### 异步编程

**核心概念**：
- `async def`：定义异步函数
- `await`：等待异步操作完成
- `asyncio`：异步 I/O 框架

**实验代码**：
```python
import asyncio

async def fetch_data(url: str):
    # 模拟异步请求
    await asyncio.sleep(1)
    return f"Data from {url}"

async def main():
    # 并发调用多个 API
    results = await asyncio.gather(
        fetch_data("api1"),
        fetch_data("api2"),
        fetch_data("api3")
    )
    print(results)

asyncio.run(main())
```

**应用场景**：
- 并发调用多个 LLM API
- 批量处理请求
- 提升 Agent 响应速度

## Day 3-4：LLM 基础概念

### Token 是什么？

**概念理解**：
- Token 是文本的最小单位
- 类比：Token ≈ 字符的"货币"

**示例**：
- 英文：`"Hello, world!"` ≈ 4 tokens
- 中文：`"你好，世界！"` ≈ 6-8 tokens（取决于分词）

**成本计算**：
- GPT-3.5 Turbo: $0.0005 / 1K tokens
- GPT-4: $0.03 / 1K tokens
- 1K tokens ≈ 750 个英文单词

**估算工具**：
- [OpenAI Tokenizer](https://platform.openai.com/tokenizer)

---

### Temperature 参数

**作用**：控制生成结果的随机性

**参数值**：
- `0.0`：确定性输出（每次相同）
- `0.7`：平衡（推荐）
- `1.0`：高随机性（创意性强）

**实验结果**：
- Temperature = 0.0：输出稳定，适合事实性任务
- Temperature = 0.7：输出平衡，适合对话
- Temperature = 1.0：输出多样，适合创意生成

---

### Context Window（上下文窗口）

**概念**：模型一次能处理的最大 Token 数

**不同模型的 Context Window**：
- GPT-3.5: 4K tokens
- GPT-4: 8K / 32K tokens
- GPT-4 Turbo: 128K tokens
- glm-5.2: 104万 tokens（超长上下文）

**影响**：
- 超过 Context Window 的内容会被截断
- 长对话需要记忆管理
- 成本与 Context Window 相关

## Day 5-7：Prompt Engineering

### Zero-shot vs Few-shot

**Zero-shot（零样本）**：
```python
prompt = "将以下英文翻译为中文：Hello, world!"
```

**Few-shot（少样本）**：
```python
prompt = """
将英文翻译为中文：
英文：Hello, world!
中文：你好，世界！

英文：How are you?
中文：你好吗？

英文：Good morning!
中文：早上好！

英文：Nice to meet you!
中文："""
```

**对比**：
- Zero-shot：简单任务，无需示例
- Few-shot：复杂任务，示例提升准确率

---

### Chain-of-Thought (CoT)

**概念**：让模型逐步推理，展示思考过程

**示例**：
```python
prompt = """
问题：小明有 5 个苹果，给了小红 2 个，又买了 3 个，现在有几个？

请一步步思考：
1. 小明开始有 5 个苹果
2. 给了小红 2 个，剩下 5 - 2 = 3 个
3. 又买了 3 个，现在有 3 + 3 = 6 个

答案：6 个苹果
"""
```

**优势**：
- 提升复杂推理任务准确率
- 错误更容易调试
- 可解释性更强
